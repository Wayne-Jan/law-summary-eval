RULE-BASED METRICS
規則式自動指標

程式自動比對日期、數字、判決結果等客觀事實，
不涉及主觀判斷，結果完全可重現。

運作方式

從摘要中擷取日期、數字、判決結果、機構名稱等客觀資訊，與原文及標準答案逐一比對。每項指標都有明確的計算公式，不依賴任何模型判斷。

📎
引用率
Citation Rate
摘要中有多少比例的陳述標註了 [CH_XX] 原文引用？引用率越高，代表摘要的每句話越有據可查。
0 – 1（比率）
計算方式
帶有 [CH_XX] 標記的陳述句數 ÷ 全部陳述句數
為什麼重要
沒有標註引用的句子無法追溯至原文，讀者無從驗證其正確性。高引用率是「可查核」摘要的基本條件。
📅
日期正確率
Date F1
摘要中提到的日期是否正確？系統將民國/西元日期統一轉換後，與標準答案雙向比對（Precision + Recall）。
0 – 1（F1 分數）
計算方式
Precision（摘要中日期的正確比率）× Recall（標準答案中日期的涵蓋比率）的調和平均
為什麼重要
醫療糾紛案件中，就醫日期、手術日期、死亡日期等時間資訊對判決認定至關重要，日期錯誤可能改變整個案件的理解。
#
數值正確率
Num F1
刑期、罰金、檢驗數值等關鍵數字是否正確？系統進行單位正規化後比對。
0 – 1（F1 分數）
檢查項目
刑期月數、緩刑年數、罰金金額、實驗室數值（如 CPK 198 U/L）等，含單位正規化
為什麼重要
「有期徒刑三年」寫成「有期徒刑二年」是嚴重的事實錯誤，數字精確度直接影響摘要的可信度。
⚖
判決結果吻合
Verdict Match
摘要是否正確記載判決結果？包含有罪/無罪、刑期長度、緩刑年數三個欄位的精確比對。
0 – 1（吻合率）
檢查欄位
verdict_type（有罪/無罪）、sentence_months（刑期月數）、probation_years（緩刑年數）
為什麼重要
判決結果是整份摘要最核心的結論，寫錯有罪/無罪或刑期是最嚴重的摘要錯誤。
🏛
機構歸屬覆蓋
Institution Attribution
法院、醫審會、醫院等關鍵機構是否都有提及？衡量摘要對案件相關單位的涵蓋程度。
0 – 1（涵蓋率）
計算方式
摘要中正確提及的機構數 ÷ 標準答案中的機構總數
為什麼重要
鑑定意見的出處（行政院衛生署醫事審議委員會 vs 法醫研究所）對判決的證據力判斷至關重要。
👻
幻覺率
Hallucination Rate (Rule)
摘要中是否出現原文完全沒有的虛構資訊？偵測不存在於原文或標準答案中的實體（人名、機構、數據）。
0 – 1（越高越好，= 1 - 幻覺比率）
計算方式
可驗證的實體數 ÷ 摘要中全部實體數。被認定為「幻覺」的實體不存在於原文或標準答案中。
為什麼重要
AI 模型有時會「憑空捏造」事實——例如虛構一個不存在的鑑定機構或錯誤的診斷名稱。這在法律文件中是絕對不可接受的。
LLM-JUDGE METRICS
大語言模型評審指標

由 Claude Sonnet 依據結構化評分標準，
從事實涵蓋率、寫作品質、引用忠實度三個面向進行深度評估。

運作方式

以結構化萃取的事實資料（時間軸、人物、論證等）作為評分基準，由 Claude Sonnet 模型逐項核對摘要內容。每個維度都有錨定量表（Anchored Likert Scale），確保評分標準一致。

事實涵蓋率 Fact Recall

AI 產生的摘要涵蓋了多少關鍵事實？以結構化萃取的 8 類事實欄位為基準，逐項確認摘要是否提及。

📆
時間線事件
Timeline Recall
案件的關鍵事件是否都有記錄？就醫、手術、併發症、死亡等時間節點。關鍵決策點加權 2 倍。
0 – 1（加權涵蓋率）
加權機制
標記為 critical_decision 的事件（如「決定轉院」「實施手術」）權重為 2 倍，確保關鍵轉折點不被遺漏。
👤
關鍵人物
Entity Recall
被告、被害人、關鍵醫療人員、鑑定機構等重要角色是否都有提及？支援匿名化對應（甲 = 戊○○）。
0 – 1（涵蓋率）
⚔
起訴指控
Allegation Coverage
檢察官的起訴要旨是否被完整記錄？包含指控的具體過失行為與法律依據。
0 – 1（涵蓋率）
🛡
被告辯護
Defense Recall
被告的辯護主張是否被記錄？包括醫療行為合理性的論述與抗辯理由。
0 – 1（涵蓋率）
🔬
鑑定意見
Expert Opinion Recall
醫審會、法醫研究所等鑑定機構的意見是否完整呈現？含核心結論、注意義務違反判斷、因果關係評估。權重 1.5 倍。
0 – 1（加權涵蓋率）
子欄位檢查
每份鑑定意見拆分為：鑑定機構、核心結論、注意義務違反評估、因果關係評估，逐項確認。
為什麼加權
鑑定意見是法院判決的核心證據依據，遺漏將嚴重影響摘要的完整性。
📋
判決理由
Reasoning Coverage
法院的認定事實、法律分析、加重/減輕量刑因素是否被涵蓋？
0 – 1（涵蓋率）
🔗
因果鏈
Causation Chain Recall
從過失行為到被害人死亡的因果推論鏈條，是否每個環節都有交代？
0 – 1（涵蓋率）
⚖
量刑正確性
Sentence Accuracy
每位被告的刑期、緩刑等量刑細節是否正確？權重 1.5 倍。
0 – 1（加權正確率）
檢查欄位
逐位被告比對：刑種（有期徒刑/拘役）、刑期月數、緩刑年數、有罪/無罪

引用忠實度 Faithfulness

摘要中每一句標記了 [CH_XX] 引用的陳述，是否真的被原文段落支持？逐句驗證，依支持程度分為 A-D 四級。

A
完全支持
核心主張明確出現在引用段落中，事實、數字、因果關係都正確
× 1.0
B
實質支持
核心主張有依據，但摘要做了合理的改寫、濃縮或推論
× 0.75
C
弱支持
引用段落提到相關主題，但具體主張無法直接從中推導
× 0.25
D
不支持
引用段落完全無相關資訊，或摘要內容與引用段落矛盾
× 0.0

最終分數 = (A×1.0 + B×0.75 + C×0.25 + D×0.0) ÷ 總評分句數
跳過無引用的句子與標題行，不計入分母。

NLP REFERENCE-BASED METRICS
參考式比對指標

將 AI 摘要與法學教授撰寫的標準摘要進行文字與語意比對，
衡量兩者的相似程度。

運作方式

這類指標不判斷「摘要對不對」，而是衡量「AI 摘要跟標準摘要有多像」。ROUGE 看文字層面的重疊，BERTScore 看語意層面的相似度。兩者互補：ROUGE 抓用詞差異，BERTScore 容忍合理的換句話說。

R
ROUGE-L
Longest Common Subsequence
AI 摘要與標準摘要之間最長共同子序列的比率。用詞越接近標準摘要，分數越高。
0 – 1（F1 分數）
優點
計算簡單、可重現、是文本摘要領域最通用的基準指標
限制
只看字面重疊，無法捕捉同義替換（如「死亡」vs「過世」），對中文分詞敏感
B
BERTScore F1
Contextual Embedding Similarity
使用預訓練語言模型計算語意向量的相似度。即使用詞不同，只要意思相近就會得高分。
0 – 1（F1 分數）
優點
能捕捉語意相似性，容忍合理的改寫與同義替換
限制
依賴預訓練模型品質，對繁體中文的支援較英文弱，且分數分佈範圍較窄（多集中在 0.6-0.8）
